Active Directory Lab Attack Simulation

July 20, 2025 5 min read Active Directory Security

This project involved designing and implementing a vulnerable Active Directory (AD) lab environment to practice and demonstrate common attack vectors used by red teams and malicious actors. The primary goal was to gain practical experience in exploitation, defense, and detection within a simulated enterprise network.

Lab Setup and Configuration

The lab comprised three virtual machines: a Domain Controller (Windows Server), a Domain Joined Client (Windows 10), and a Kali Linux attack machine. Key services like DNS, DHCP, and Kerberos were configured to mirror a realistic small business environment.

Attack Scenarios Explored

We successfully executed several key attack techniques:

  1. **Kerberoasting:** Exploiting service principal names (SPNs) to harvest password hashes.
  2. **AS-REPRoasting:** Targeting user accounts configured not to require Kerberos preauthentication.
  3. **Pass-the-Hash (PtH):** Utilizing NTLM hashes extracted from memory to authenticate without cracking the password.
  4. **BloodHound Mapping:** Using BloodHound and SharpHound to map privilege escalation paths and identify vulnerable objects within the domain structure.

Mitigation Strategies

For each attack vector, corresponding defensive measures were researched and implemented:

  • **Kerberoasting Mitigation:** Enforcing complex, long passwords for service accounts and ensuring sensitive service accounts are protected by Protected Users Security Group.
  • **AS-REPRoasting Mitigation:** Disabling the "Do not require Kerberos preauthentication" option on all standard user accounts.
  • **Detection:** Configuring Windows Event Logging (specifically event IDs 4768 and 4769) and utilizing a simulated SIEM (Splunk or ELK stack) to detect abnormal Kerberos requests and suspicious logon activity.

Conclusion and Key Takeaways

This lab confirmed the significant security risks posed by common AD misconfigurations. The hands-on experience in both attacking and defending solidified the importance of proper privilege management, strong password policies, and continuous monitoring (Blue Team operations) for domain integrity. The knowledge gained is directly applicable to penetration testing and enterprise security roles.
